# Tópico 1 - Preparação do ambiente Linux

## Ambiente utilizado
O ambiente utilizado foi o WSL (Windows Subsystem for Linux) no Windows, permitindo executar Linux de forma integrada no sistema operativo Windows.


## Objetivo desta atividade
O objetivo desta atividade foi:
- Preparar o ambiente Linux para desenvolvimento
- Familiarizar-me com o terminal
- Criar a estrutura inicial do projeto
- Registar comandos essenciais utilizados no Linux

## Comandos executados
Durante esta atividade foram utilizados os seguintes comandos:

- ls: listar ficheiros e diretórios
- cd: navegar entre diretórios
- pwd: mostrar o diretório atual
- mkdir: criar pastas
- rm: remover ficheiros ou diretórios
- tree: visualizar estrutura de pastas em árvore
- git init: inicializar repositório Git
- git status: verificar estado do repositório
- git commit: registar alterações no repositório


## Diferença entre VM, VPS e Cloud

- VM (Máquina Virtual): ambiente virtual local que simula um sistema operativo dentro do computador. 
- VPS (Virtual Private Server): servidor virtual remoto usado para alojar aplicações ou websites.
- Cloud (Infraestrutura em nuvem): conjunto de serviços remotos escaláveis (armazenamento, servidores e aplicações) fornecidos por plataformas como AWS, Azure ou Google Cloud.
